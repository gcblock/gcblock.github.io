# In this paper, the design and implementation of meta-metadata system are introduced

# The introduction

Metadata is defined as: data that describes data, and descriptive information about data and information resources.

Metadata is data about other data, or structured data used to provide information about a resource.Metadata is the data that describes the object such as information resource or data. Its purpose is: to identify the resource; Evaluation resources; Track changes in the use of resources; To achieve simple and efficient management of a large number of network data; To realize the effective discovery, search, integration organization and effective management of information resources. The basic characteristics of metadata are as follows:

Metadata can build a data model, a logical framework. Because metadata is also data, it can be stored and obtained in the database in a similar way, and it is easy to spread and share. If the organization that provides the data metadata also provides the metadata that describes the data metadata, the use of the data metadata will be accurate and efficient. When users work with data, they can first look at its metadata so that they can get the information they need.

The concept of metadata can not be further described here.Here share under the present excellent metadata framework RDFS, relevant API with Apache Jena, and RDF data handler rdfgui, although the metadata framework of Yale university and oracle's design is very good, but it is not as a WEBAPP framework to research and development, but from the view of scientific development of experimental architecture, no combined with years of mature Internet application technology research and development, thus losing popularity, but also because he is the pioneer to encourage the subsequent developers involved, like COINS on the creation of block chain technology,, salute!

Nowadays, the concept of metadata has become popular, and more and more software development companies begin to feel that adding, deleting, changing and checking in the business are repeating the wheel. Moreover, complex models coupled to the code are tightly coupled to the business logic, and software iteration and business modification are costly to develop.Therefore, A-Meta metadata system is committed to solving these problems from the point of pain in network applications.

# 2 Design and implementation of the system

Relational database is an important technology of Internet application, which has been practiced for more than 20 years. It can withstand the test of business and the query speed of big data.Today we use four tables to implement the persistence layer in the metadata system architecture, two tables to define the model, and two tables to store the entity data.

# 2.1 Description table of the model

According to the Taoist thought of heaven and earth, every object is made up of the most basic atoms, and the spatial structure represents different substances according to different amounts of aggregation. In this system, both classes and attributes are composed of metadata. Which properties belong to which class is stored by the relational tables of the model.

# 2.3 Entity data table

The data structure of the entity data table is the mapping of the description table, which is used to hold the entity data that describes the model structure defined by the table.

CREATE TABLE `instance` (
    'id' bigint(20) NOT NULL COMMENT 'primary key ', 
    'value' varchar(1000) NOT NULL COMMENT 
    Description_id 'bigint(20) NOT NULL COMMENT' description_id ' 
    PRIMARY KEY (`id`) 
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

# 2.4 Data Relation Table

The data structure of the entity data relational table is the mapping that describes the relational table and is used to store the entity data association relationship.

CREATE TABLE `instance_relation` (
    'id' bigint(20) NOT NULL COMMENT 'primary key ', 
    'instance_id' bigint(20) NOT NULL COMMENT 'instance_id ', 
    Parent_instance_id 'bigint(20) NOT NULL COMMENT' parent_instance_id ', 
    PRIMARY KEY (`id`),
    KEY `instanceId` (`instance_id`), 
    KEY `parentInstanceId` (`parent_instance_id`) 
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

# 2.6 Two-dimensional data query

In the actual production, people contact the most is the two-dimensional data body, such as Excel sheet, sheet is the indication, the top of the table is the attribute, the bottom is the data.The structure is easy to understand. So how do we construct an SQL statement to query a piece of user data?

SELECT
	User.instance_id AS user Id,
	'value' AS user name
FROM
	(
		SELECT
			 instance.id AS instance_id, 
			instance.`value`
		FROM
			instance
		WHERE
			instance.description_id = 1001
	 ) AS the user 
LEFT JOIN (
	SELECT
		instance_relation.parent_instance_id,
		instance_relation.instance_id,
		instance.`value`
	FROM
		instance_relation
	 LEFT JOIN instance ON instance_relation.instance_id = instance.id 
	WHERE
		instance.description_id = 1002
AS username ON user.instance_id = username.parent_instance_id

# 2.8 Executable scripts in the process of adding, deleting, modifying and checking

The A-Meta system is compatible with complex business logic by executing scripts when data is added, deleted, modified or searched.The A-Meta system is developed in Java and the scripting engine is Groovy.https://baike.baidu.com/item/Groovy/180590?fr=aladdin (opens new window)The system exposes the API through Feigin, and the Groovy script can directly call related services to operate the data.
